meetupService
service in module app

Description

The meetupService provides abstraction on data source and allows to perform basic operations on members. It also substitutes default picture using noPhotoUrl constant and bio for members that don't have this information.

Internally meetupService heavily relies on app.refService that provides low-level data access API.

Dependencies

  • $firebaseArray
  • refService
  • noPhotoUrl

Methods

  • getMember(member)

    Returns member object by it's ID

    meetupService.getMember('1');
    Parameters
    ParamTypeDetails
    memberStringNumber

    ID. String value will be parsed as integer before use.

  • getMembers()

    Returns an object containing list of members. First time method is called, the list of members will be empty and will be populated asynchronously once data received from refService.

    meetupService.getMembers();

  • updateMember(member)

    In is not necessary to pass all properties of a member, only those that have been changed. Only the presance of $id property in the object is mandatory, all the others are optional.

    Member object that is passed as parameter will be cloned and all properties starting from one of the following characters will be deleted: .#$/[].

    meetupService.updateMember({
    $id: 1,
    name: 'John Doe',
    city: 'Tel Aviv',
    notes: 'Some useful notes'
});
    Parameters
    ParamTypeDetails
    memberObject

    object representing a meetup member
